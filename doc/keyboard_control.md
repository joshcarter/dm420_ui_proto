# Keyboard Control

Keyboard control will be necessary for this application. We will use a
user-selectable keyboard shortcut to select which panel is receiving
keyboard input. Default key bindings might be command+1 for the first
panel, command+2 for the second, and so on.

The behavior of each panel will be based on that panel's role. For
example, the "band scan" panel might only have one control: start or
cancel a scan.

The "waterslide" panel will be more sophisticated, as it needs the
keyboard for both sending messages and changing radio settings.
Further detail on each panel's behavior will be covered in documents
describing that panel.
